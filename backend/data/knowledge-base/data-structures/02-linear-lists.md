# 数组与链表
## 线性表定义
线性表是具有有限个数据元素的有序序列，除首尾元素外每个元素都有唯一前驱与后继。数组和链表是线性表的两种存储实现，不是两个不同的逻辑模型。
## 数组性质
数组使用连续地址存储同类型元素，可依据基址和元素宽度计算下标位置，因此随机访问为 O(1)。连续布局通常具有良好缓存局部性。
## 数组插入删除
在数组中间插入要为新元素腾出位置，删除要填补空位，最坏均移动 O(n) 个元素。尾部追加在容量足够时为 O(1)，动态扩容时使用摊还分析。
## 单链表性质
单链表结点保存数据与后继指针，结点无需连续。它不能常数时间访问第 i 项，但已知前驱后能以 O(1) 修改链接完成插入或删除。
## 双向链表
双向链表增加前驱指针，支持从任一结点向两侧移动，也能在已知结点时直接删除。代价是每个结点更多空间和更多指针一致性维护。
## 头结点与哨兵
不承载业务数据的头结点可统一空表与首元结点操作，减少分支。哨兵不是有效元素，长度统计和遍历终止条件不能把它算入数据。
## 反转链表
迭代反转维护 previous、current、next 三个引用：先保存后继，再改当前链接，最后整体前移。顺序错误会丢失尚未遍历的链段。
## 示例
维护频繁在两端增删的序列时可选双向链表；需要大量按下标读取且长度较稳定时数组更合适。选择依据是操作分布，而不是笼统判断谁更快。
## 复杂度
数组读取 O(1)、按值查找 O(n)、中间增删 O(n)。单链表读取第 i 项 O(n)、按值查找 O(n)、已知前驱的增删 O(1)。
## 常见错误
链表删除时忘记释放结点会泄漏内存，释放后继续访问会产生悬空指针。修改双向链表时只更新一个方向，会破坏前后链接一致性。

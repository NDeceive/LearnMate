# 红黑树

## 定义与用途

红黑树是一棵二叉搜索树，每个内部结点额外保存红色或黑色。颜色约束并不是为了让树完全平衡，而是限制任意根到叶路径的长度差，使查找、插入和删除的最坏时间复杂度保持在 O(log n)。它适合需要频繁动态更新、同时要求最坏性能上界的有序集合与映射。

红黑树必须继续满足二叉搜索树次序：任一结点左子树中的键小于该结点，右子树中的键大于该结点。旋转只改变局部父子关系，不改变中序遍历的键序列。

## 五条性质

每个结点是红色或黑色；根结点为黑色；所有空叶哨兵 NIL 为黑色；红结点的两个孩子均为黑色；从任一结点到其后代 NIL 的所有简单路径包含相同数量的黑结点。最后一项中的数量称为该结点的黑高。

“红结点不能相邻”来自第四条性质，但它不能替代全部性质。检查修复结果时还要确认根为黑色、NIL 为黑色，并确认所有路径黑高相同。

## 左旋

对结点 x 左旋的前提是 x 的右孩子 y 不是 NIL。操作先把 y 的左子树接为 x 的右子树；如果这棵子树非空，更新它的父指针为 x。随后让 y 接替 x 在原父结点下的位置，再把 x 设为 y 的左孩子并更新 x 的父指针。

左旋前局部中序关系为 A、x、B、y、C，其中 A 是 x 的左子树，B 是 y 的左子树，C 是 y 的右子树。左旋后仍为 A、x、B、y、C，所以二叉搜索树的中序顺序不变。常见错误是先覆盖 x.right，之后才尝试取得 y，或忘记更新 B 的父指针。

## 右旋

右旋与左旋镜像。对结点 y 右旋要求其左孩子 x 不是 NIL：把 x 的右子树接为 y 的左子树，维护该子树父指针；让 x 接替 y 的原位置；最后令 y 成为 x 的右孩子。

右旋前后的局部中序关系都为 A、x、B、y、C。旋转本身不自动恢复颜色性质，调用者必须根据插入或删除修复的具体分支完成变色。

## 插入修复

新结点按普通二叉搜索树规则插入并着红色。若父结点为黑色，不会破坏红黑性质；若父结点为红色，则出现连续红结点。修复围绕父结点、祖父结点和叔叔结点展开。

叔叔为红色时，把父和叔叔改为黑色、祖父改为红色，再从祖父继续向上检查。叔叔为黑色时，根据新结点、父结点与祖父结点形成直线还是折线，先把折线旋为直线，再围绕祖父旋转并交换相关颜色。循环结束后必须把根设为黑色。

## 删除修复

删除黑结点可能让一条路径少一个黑色，因此需要修复“双黑”位置。算法依据兄弟的颜色、兄弟两个孩子的颜色以及双黑结点位于左侧还是右侧分类；左右分支互为镜像。

兄弟为红色时先对父结点旋转并交换父兄颜色，把问题转成黑兄弟情形。兄弟为黑色且两个孩子都黑时，可把兄弟变红并把缺少的黑色向父结点上传。若兄弟存在红孩子，则通过一次或两次旋转与变色消除黑高差。实现时 NIL 哨兵也必须具有可用的颜色语义。

## 与 AVL 树的区别

AVL 树直接约束左右子树高度差，通常查找路径更短，但更新时可能需要更严格的平衡维护。红黑树约束路径黑高并禁止连续红结点，平衡较宽松，动态插入删除通常只需常数次旋转。两者的查找、插入、删除最坏复杂度都是 O(log n)。

不能简单断言某一种树在所有场景更快。读多写少且查找延迟敏感时可考虑 AVL；更新频繁、需要成熟有序容器实现时红黑树常见。选择应结合常数开销、实现复杂度和工作负载验证。

## 常见错误与检查清单

常见错误包括混淆左旋与右旋的提升孩子、遗漏父指针、把 NIL 当作普通空值而无法读颜色、修复时使用已经变化的亲属关系、只检查红结点相邻而不检查黑高，以及忘记最终把根染黑。

验证实现时应同时检查二叉搜索树中序有序、根和 NIL 为黑色、红结点孩子为黑色、每个结点到后代 NIL 的黑高一致。随机插入删除测试应在每一步执行这些不变量检查，而不是只比较最终查找结果。

## 操作复杂度

红黑树高度为 O(log n)，因此查找、插入和删除最坏均为 O(log n)。插入修复最多执行常数次旋转，删除修复也只需常数次旋转，但变色与向上检查可能经过 O(log n) 层。

旋转本身只更新固定数量的父子链接，时间和额外空间均为 O(1)。复杂度结论依赖所有更新完成后继续满足红黑性质，不能把未修复的普通搜索树当作具有相同上界。

## 小型示例

若局部中序键为 10、15、20，结点 10 的右孩子为 20，而 20 的左孩子为 15，对 10 左旋后 20 上升、10 成为其左孩子、15 成为 10 的右孩子。中序仍为 10、15、20。

这个例子说明中间子树必须转接，而不是简单交换两个结点。真实实现还要维护它们与原祖父结点的链接；若旋转的是根，则树的根引用必须更新为 20。

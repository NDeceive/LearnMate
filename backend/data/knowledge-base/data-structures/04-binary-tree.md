# 树与二叉树
## 树定义
树由结点和层次关系组成，非空树有唯一根，其余结点划分为互不相交的子树。结点的度是孩子数量，树的高度需明确从零还是从一计数。
## 二叉树性质
二叉树每个结点最多有左、右两个有序孩子。第 i 层最多有 2 的相应幂个结点，含 n 个结点的非空二叉链表有 n+1 个空孩子位置。
## 满树与完全树
满二叉树每层都达到最大结点数；完全二叉树只允许最后一层未满，并且结点从左到右连续。完全二叉树适合使用数组紧凑存储。
## 先序遍历
先序按照根、左、右访问，适合输出层次结构的前缀表示或复制树。递归实现先处理当前结点，再递归两个子树。
## 中序遍历
中序按照左、根、右访问。二叉搜索树的中序结果按键递增，这是搜索树次序约束的直接结果，普通二叉树不保证有序。
## 后序遍历
后序按照左、右、根访问，适合先处理孩子再处理父结点的任务，例如释放整棵树、计算目录占用或求子树汇总值。
## 层序遍历
层序使用队列：根入队，循环取出一个结点并把非空孩子入队。若要区分层，可记录每轮开始时的队列长度。
## 递归与迭代
递归遍历简洁但调用栈深度与树高相关；极度偏斜的树可能造成栈溢出。迭代遍历显式维护栈或队列，状态转移更容易监控。
## 复杂度
四种完整遍历都访问每个结点一次，时间为 O(n)。额外空间取决于树高或最大层宽：深度遍历为 O(h)，层序最坏为 O(n)。
## 常见错误
混淆遍历名称、漏写空指针终止条件和把普通二叉树中序误认为有序是常见错误。构造树时还要区分左右孩子顺序，不能把二叉树当无序树。

# 复杂度与算法分析
## 定义
算法复杂度描述资源消耗随输入规模增长的趋势。时间复杂度统计基本操作次数的渐近量级，空间复杂度统计额外存储需求，不等同于某台机器上的实际秒数或字节数。
## 渐近记号
O 给出渐近上界，Ω 给出渐近下界，Θ 同时给出紧确上下界。使用记号时必须说明输入规模和被计数操作，不能把 O(n) 直接解释成一定执行 n 次。
## 最好最坏与平均情况
同一算法在不同输入上可能有不同代价。最坏情况提供确定上界，平均情况依赖输入分布，摊还分析则把一串操作的总代价分摊到单次操作。
## 顺序结构操作
数组按下标访问通常为 O(1)，但在中间插入需要移动后继元素而为 O(n)。链表已知结点后插入可为 O(1)，先定位目标仍可能需要 O(n)。
## 递归分析
递归算法需要把当前层工作量与子问题代价一起写成递推式。例如归并排序满足 T(n)=2T(n/2)+O(n)，得到 Θ(n log n)。
## 空间复杂度
空间分析应区分输入本身与额外空间。递归调用栈、临时数组、辅助哈希表都属于额外空间；原地算法通常只使用常数或对数级辅助空间。
## 摊还分析
动态数组扩容一次需要复制多个元素，但容量按倍数增长时，一系列 append 的总复制次数为线性量级，所以单次追加的摊还复杂度为 O(1)。
## 示例
在长度 n 的无序数组中顺序查找目标，最坏比较 n 次，时间为 O(n)。若数组有序且支持随机访问，可用二分查找把最坏比较次数降为 O(log n)。
## 常见错误
忽略隐藏常数会影响小规模性能，但不能据此否定渐近分析。另一个错误是只看循环层数：循环边界可能按倍数变化，单层循环也可能是 O(log n)。
## 选择依据
复杂度是选择结构的重要依据，但还要考虑缓存局部性、实现常数、稳定性和工作负载。理论同阶的两个实现，在真实数据上仍应通过基准测试比较。

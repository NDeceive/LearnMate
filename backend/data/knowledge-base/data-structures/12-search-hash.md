# 查找与哈希表
## 顺序查找
顺序查找逐项比较，适用于无序序列和只支持顺序访问的结构。最坏 O(n)，可用哨兵减少循环中的边界判断。
## 二分查找前提
二分查找要求数据按比较规则有序并支持有效的中间位置访问。链表虽然有序，但定位中点不是 O(1)，通常不适合标准二分查找。
## 二分边界
闭区间写法维护 low≤high，半开区间维护 low<high。循环条件、mid 更新和返回语义必须匹配，否则容易死循环或漏掉边界元素。
## 哈希表定义
哈希表用散列函数把键映射到桶位置，目标是让键均匀分布。它保存的是键值关联，不维持键的整体有序关系。
## 散列函数
散列函数对同一键必须稳定，并尽量利用键的全部信息。最终桶下标通常还要按容量取模，不能把语言运行时哈希值直接当数组下标。
## 冲突
有限桶映射大量可能键必然产生冲突。正确实现必须使用链地址法或开放定址法等策略，不能假设好函数可以彻底消除冲突。
## 链地址法
每个桶保存冲突元素容器，装载因子可超过一。查找先定位桶再在桶内比较完整键，不能只比较哈希值。
## 开放定址法
开放定址把元素都放在表数组中，通过线性、二次或双重散列探测空槽。删除通常使用墓碑标记，直接清空会截断后续查找链。
## 复杂度
分布良好且装载因子受控时查找、插入、删除平均 O(1)，最坏仍为 O(n)。扩容需要重新散列现有元素，不能简单复制到相同下标。
## 常见错误
可变对象作为键、相等键产生不同哈希、忽略装载因子、开放定址删除断链，以及把平均 O(1) 宣称为最坏 O(1) 都是错误。

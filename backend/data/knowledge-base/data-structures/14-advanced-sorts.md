# 高级排序
## 快速排序
快速排序选择枢轴并把元素划分到两侧，再递归处理子区间。划分完成后枢轴处于最终位置或区间满足进一步递归的分隔条件。
## 快排划分
Hoare 与 Lomuto 划分的指针含义和返回边界不同，递归区间必须与实现匹配。混合两套边界规则常导致漏排或无限递归。
## 快排复杂度
平衡划分时为 O(n log n)，持续极端划分时最坏 O(n²)。随机枢轴或三数取中可降低特定输入触发最坏形态的概率。
## 归并排序
归并排序把序列分成两半分别排序，再用双指针线性合并。时间稳定为 O(n log n)，数组实现通常需要 O(n) 辅助空间。
## 归并稳定性
合并时相等键优先取左半部分即可保持原相对顺序。若相等时优先取右侧，会破坏稳定性但不影响键值有序。
## 堆排序
堆排序先建立最大堆，再反复把堆顶交换到未排序区尾部并恢复堆序。最坏 O(n log n)、原地，但通常不稳定。
## 排序比较
快排平均性能和缓存局部性较好；归并适合稳定排序与外部排序；堆排序提供原地最坏 O(n log n) 上界。选择应依据约束。
## 示例
归并两个有序段 1、4、7 与 2、3、8，双指针依次选 1、2、3、4、7、8，每个元素只进入结果一次。
## 递归空间
平衡快排递归栈 O(log n)，最坏 O(n)；可先递归较小区间并循环处理较大区间以限制栈深。归并递归栈另有 O(log n)。
## 常见错误
快排递归包含枢轴导致区间不缩小、归并忘记复制剩余段、堆排序把有效堆长度与数组长度混用，都是高频错误。

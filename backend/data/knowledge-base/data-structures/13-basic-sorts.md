# 基础排序
## 排序目标
排序按给定比较规则重排记录。除键顺序外，还要关注稳定性、额外空间、输入分布以及数据是否支持随机访问。
## 插入排序
插入排序维护有序前缀，把当前元素向前移动到合适位置。时间最坏 O(n²)，近乎有序时接近 O(n)，通常稳定且原地。
## 选择排序
选择排序每轮从未排序区选择最小项放到边界。比较次数始终为 Θ(n²)，交换次数较少，直接交换实现通常不稳定。
## 冒泡排序
冒泡排序比较相邻逆序对并交换，使极值逐轮到达末端。设置本轮无交换标记可让有序输入提前结束。
## 稳定性
稳定排序保持相等键记录的原相对顺序。稳定性对多关键字分阶段排序有意义，但只比较整数值时不一定能从输出观察到。
## 插入示例
序列 2、5、7、3 的有序前缀为 2、5、7，把 3 暂存后移动 7 和 5，再写入 2 后方，得到 2、3、5、7。
## 冒泡示例
对 3、1、2 第一轮比较 3 与 1 并交换，再比较 3 与 2 并交换，最大值 3 到达末端；下一轮只需处理前两项。
## 复杂度比较
三种基础排序最坏时间均 O(n²)，额外空间 O(1)。插入和优化冒泡在已有序输入可达 O(n)，选择排序仍需完整寻找最小值。
## 使用场景
小规模数组或作为高级算法的小区间优化时，插入排序常因常数小和局部性好而有价值。选择排序适合交换代价远高于比较的特殊场景。
## 常见错误
循环边界多比较一次、覆盖待插入值、把选择排序误判稳定，以及冒泡每轮仍扫描已就位尾部会造成错误或多余工作。
